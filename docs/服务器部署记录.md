## Nginx 配置

路径：`/etc/nginx/sites-available/yijiahu`

```nginx
server {
    listen 80;
    server_name _;
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
    }
}
```

## systemd 服务

路径：`/etc/systemd/system/yijiahu.service`

```ini
[Unit]
Description=Yijiahu Backend API
After=network.target postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/yijiahu/yijiahu/backend
EnvironmentFile=/opt/yijiahu/yijiahu/backend/.env.prod
ExecStart=/usr/bin/node dist/main.js
Restart=always
RestartSec=10
StandardOutput=append:/opt/yijiahu/yijiahu/logs/app.log
StandardError=append:/opt/yijiahu/yijiahu/logs/app.log

[Install]
WantedBy=multi-user.target
```

## 踩坑记录

| 问题 | 解决 |
|------|------|
| `@nestjs/typeorm` crypto.randomUUID Node 18 不支持 | 升级 Node.js 18 → 20 |
| TypeORM synchronize 在 production 被硬编码禁用 | 临时改 NODE_ENV=development 创建表后改回 |
| SSH 密码认证被禁用 | 导入 SSH 公钥到阿里云 |
| 数据库用户 sudo -u postgres 无权限 | 使用 `PGPASSWORD=xxx psql -U yijiahu -h localhost` |
| 阿里云安全组未开放 80 端口 | 阿里云控制台 → 安全组 → 入方向规则 |
